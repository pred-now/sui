# The Capital Rule

Two hard limits make the whole design safe: the imbalance ceiling and the capital
rule. Both are enforced before an open is accepted, not as a setup calculation.

## Where 0.14 comes from

Pred simulated the worst pool loss per unit of peak leveraged open interest, over
20,000 single-market runs including black swans.

| loss / OI | p99 | p99.9 | p99.99 |
| --- | --- | --- | --- |
| normal | 0.070 | 0.110 | 0.148 |
| jump | 0.085 | 0.136 | 0.166 |

The rule takes the conservative p99.9:

> Hold pool plus treasury at least **0.14x** peak leveraged open interest.
> Equivalently, cap leveraged OI at about **7x** the pool.

In simulation, sizing the pool to 0.14x median OI drops single-market insolvency
from about 0.5% to about 0.04%.

## The admission check

Every open is checked in the engine before any signing.

```ts
projImb = imbalance after this open
projOI  = peak leveraged OI for the backing pool after this open

if (projImb <= ceiling && projOI <= pool / 0.14) {
    accept()
} else if (reducesImbalance(open)) {
    accept()                  // balancing opens are always allowed
} else {
    refuse()
}
```

The ceiling is 20%, well below the roughly 34% break point. See
[The Safety Wall](/economy/safety-wall) for the three-layer margin.

## Correlation cohorts

One subtlety. The 0.14 ratio was derived from single-market tails. Markets that
resolve on the same real-world event jump together and do not diversify.

> For a group of correlated markets backed by one pool, gate on the summed cohort
> open interest, not per market. Independent, staggered markets can share a pool
> with a smaller aggregate buffer because their tails are independent.

So each market is tagged with a correlation cohort id. The gate sums OI within a
cohort and applies 0.14 to that sum. A market defaults into its own cohort unless
explicitly marked independent. Conservative by default, which is what you want for
the rule that keeps Pred solvent.

## The invariants

These hold at all times, checked continuously:

```ts
poolAssets >= 0.14 * leveragedOI        // opens and withdrawals both gated on this
hot        >= badStretchFloor           // survive without waiting on a Margin withdrawal
sumShareValues == poolAssets            // share accounting always closes
marginBorrowPath == none                // Pred only supplies its own funds
feesAndLosses counted once per eventId  // a retry never double counts
```

These are not aspirations. They are re-proven by the
[validation harness](/technical/harness) on every parameter change.
